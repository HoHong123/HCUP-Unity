# HUI Changelog

## [1.0.0] - 2026-03-31

### 주요 변경
- 버튼, 토글, 드롭다운, 팝업, 패널, 스크롤뷰 계층을 전반적으로 확장했습니다.
- `HLogConsole` 기반 디버그 콘솔 UI를 추가해 런타임 로그 표시 흐름을 보강했습니다.
- `BaseRecycleView`, `GridRecycleView`, `SpanningGridRecycleView`, `VlgRecycleView` 등 재활용 스크롤뷰 구성을 추가했습니다.
- `HImage`와 전용 에디터(`HImageEditor`)를 추가해 UI 이미지 제어와 에디터 편의성을 보강했습니다.
- 버튼/토글 계열에 상태 변화용 이펙트 컴포넌트를 추가했습니다.
- 샘플 콘텐츠를 확장했습니다. `Button`, `Console`, `Dropdown`, `Popup`, `Scrollview` 샘플 씬과 프리팹 구성을 추가했습니다.

### 마이그레이션/주의
- 재활용 스크롤뷰는 셀 생성 비용과 데이터 갱신 빈도를 먼저 정리한 뒤 적용하십시오.
- 팝업/패널은 중첩 호출과 닫힘 타이밍 충돌을 방치하면 상태가 쉽게 꼬입니다. 매니저 진입점을 통일하십시오.
- 버튼/토글 이펙트는 단순해 보여도 레이아웃 갱신 비용을 유발할 수 있으므로, 대량 배치 화면에서는 프로파일링이 선행되어야 합니다.
- 샘플(`Samples~`)은 참고용입니다. 실제 배포 빌드에는 포함하지 않거나 별도 패키지로 분리하십시오.

### 검증 체크리스트
- Scrollview: 대량 데이터, 빠른 갱신, 셀 재사용 시 레이아웃 리빌드 비용과 셀 상태 오염 여부를 점검하십시오.
- Popup/Panel: 연속 호출, 닫힘 중 재진입, 애니메이션 취소 시 상태가 꼬이지 않는지 검증하십시오.
- Button/Toggle: 입력 버퍼링, 상태 이펙트, 선택 전환 타이밍이 타 프레임에서 누락되지 않는지 확인하십시오.
- DebugConsole: 로그 누적량 증가 시 재활용 리스트와 필터링 동작이 정상 유지되는지 확인하십시오.

## [0.2.1] - 2025-11-27

### 주요 변경
- 리포지토리 명칭을 **HCUP-Unity**로 업데이트하고 패키지 디렉터리 기준을 `HoHong123/HUI`로 명확히 했습니다.
- README에 최신 UPM 설치 URL을 추가해 새 저장소/폴더 구조를 반영했습니다.

### 마이그레이션/주의
- UPM 설치 시 `HCUP-Unity` 저장소와 `HoHong123/HUI` 경로를 사용하십시오.

## [0.2.0] - 2025-09-18

### 주요 변경
- 중복 스크립트를 제거하고 런타임 구성을 49개로 재정렬했습니다.
- 패키지 버전을 0.2.0으로 상향해 HUtil/HGame과 버전 일관성을 맞췄습니다.
- 카테고리별 파일 수와 샘플 분리 정보를 README에 추가했습니다.

### 마이그레이션/주의
- Canvas Rebuild/Repaint 비용을 줄이도록 빈번한 UI 갱신은 배치하거나 프레임 분산을 고려하십시오.
- DoTween/UniTask/Odin/TMP 의존성을 사전에 설치하십시오.
- 샘플(`Samples~`)은 참고용이며 실제 빌드에서는 제외하거나 네임스페이스를 프로젝트 규칙에 맞게 정리하십시오.

### 검증 체크리스트
- Scrollview/DropDown: 대량 데이터/빠른 갱신 시 레이아웃 리빌드 비용 측정.
- Popup/Panel: 애니메이션·상태 전환 중 중첩 호출/취소 경계 테스트.
- Button/Toggle: 입력 버퍼링·상태 이펙트가 타 프레임에 누락되지 않는지 확인.
